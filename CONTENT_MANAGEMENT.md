# 711Club官网内容管理指南

## 概述

711Club官网采用动态内容管理系统，您可以通过添加Markdown文件来更新网站内容。系统会自动扫描内容文件夹并生成索引，无需手动配置。

## 目录结构

```
public/content/
├── index.json              # 自动生成的内容索引（请勿手动编辑）
├── announcements/          # 活动公告文件夹
│   ├── 2024-fall-meetup.md
│   └── new-member-welcome.md
└── experiences/            # 经验分享文件夹
    ├── 保研/
    │   └── graduate-school-experience.md
    ├── 就业/
    │   └── internship-experience.md
    ├── 留学/
    ├── 实习/
    ├── 考研/
    └── 技术栈/
```

## 内容类型

**文件位置**: `public/content/announcements/`

**Front Matter格式**:
```yaml

date: "2024-11-15"

location: "东南大学九龙湖校区学生活动中心"
```

## 更新流程

### 1. 添加新内容

1. **创建Markdown文件**:
   - 活动公告：放在 `public/content/announcements/` 文件夹
   - 经验分享：放在 `public/content/experiences/[分类]/` 文件夹

2. **编写Front Matter**:
   - 按照上述格式填写元数据
   - 确保必填字段完整

3. **编写正文内容**:
   - 使用标准Markdown语法
   - 支持标题、列表、代码块、链接等

### 2. 重新构建网站

添加新内容后，需要重新构建网站以更新内容索引：

```bash
# 进入项目目录
cd 711club-website

# 重新构建（会自动生成内容索引）
pnpm run build

# 或者只生成内容索引
pnpm run build:content
```

### 3. 部署更新

构建完成后，将 `dist` 文件夹内容部署到您的服务器或GitHub Pages。

## 最佳实践

### 1. 文件命名规范

- **活动公告**: 使用描述性名称，如 `2024-fall-meetup.md`
- **经验分享**: 使用文章标题或关键词，如 `graduate-school-experience.md`

### 2. Front Matter建议

- **title**: 简洁明了，突出重点
- **excerpt**: 100-150字符，吸引读者
- **tags**: 3-5个相关标签，便于搜索
- **date**: 使用 YYYY-MM-DD 格式

### 3. 内容编写建议

- **结构清晰**: 使用标题层级组织内容
- **实用性强**: 提供具体、可操作的建议
- **格式规范**: 保持Markdown格式的一致性
- **字数适中**: 建议1000-3000字，重要内容可适当增加

### 4. 图片处理

- **图片文件**: 放在 `public/images/` 文件夹
- **引用方式**: 使用相对路径，如 `/images/example.jpg`
- **大小控制**: 建议单张图片不超过500KB

## 故障排除

### 1. 内容不显示

**检查项目**:
- Markdown文件是否放在正确位置
- Front Matter格式是否正确
- 是否重新构建了网站

**解决方案**:
```bash
# 检查内容索引是否生成
cat public/content/index.json

# 重新生成内容索引
node scripts/generate-content-index.js
```

### 2. 格式问题

**常见问题**:
- Front Matter字段名错误
- Markdown语法错误
- 文件编码问题

**解决方案**:
- 使用UTF-8编码保存文件
- 检查Front Matter字段名拼写
- 验证Markdown语法

### 3. 性能优化

**建议**:
- 控制单个文件大小
- 避免过多的图片引用
- 定期清理过期内容
